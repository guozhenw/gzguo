11.1序列化、代理

java中的序列化、代理

1. 序列化

  1. 序列化就是为了跨平台存储/网络传输的时候制定一个规则，在传输过去的时候就可以根据规则正确的反序列化出一个对象（把对象还原回来）
  2. 把对象拆成二进制数的时候，给拆出来的东西排序的过程就叫序列化。把二进制数还原成对象的时候根据当时拆的顺序还原对象的过程就叫做反序列化。
  3. 序列化是一种规则，既然是规则就有很多人来定义规则，常见的序列化方式有：JDK（不支持跨语言）、JSON、XML、Hessian、Kryo（不支持跨语言）、Thrift、Protostuff、FST（不支持跨语言）
  4. 选择协议的一些标准：协议是否支持跨平台、序列化的速度、序列化出来的大小
  5. java怎么实现序列化：实现Serializable接口

序列化常见的问题：

  • static属性不能被序列化。原因：序列化保存的是对象的状态，静态变量属于类的状态，因此 序列化并不保存静态变量。
  • Transient：短暂的。这个关键词修饰的属性一般是 银行卡号、密码等的不想被序列化的属性。所以被Transient修饰的属性不可以被序列化
  • 序列化版本号：这个版本号就和软件的版本号一样，你的软件版本号和官方的服务器的版本不一致的话，就会提醒你更新。主要的作用就是提示用户进行更新操作。我们的对象发生了变化之后（新增、修改、删除了一些属性，在修改之后反序列化就会出错），发生变化之后为了使反序列化同步得到最新的对象，我们就使用了版本号来告诉反序列化对象发生了修改，需要同步修改！
    • 工具生成的版本号：工具生成的版本号，只要你的产品发生了一点点的变化就会发生变化。用户不更新就没法玩了
    • 自定义的版本号：自定义版本号的话，修改之后，开发者可以选择不发布，那么反序列化的时候还是之前的对象，不会影响到用户使用
    • 大多数情况下，用户可以选择不更新，不更新只是无法体验新加的功能
  • 父类和子类的序列化，向下传递。父类序列化，子类就序列化。子类序列化。父类不会序列化。

2. 代理

  1. 生活中的代理：代购、中介、商家
  2. 代理模式：客户类真正想访问的是目标类，但是真正可以访问到的类确是代理类。客户类对目标对象的访问是通过访问代理对象来实现的。（代理类和目标类要实现同一个接口）

有A、B、C三个类，A原来可以调用C类的方法，现在因为某种原因C类不允许A类调用其方法，但B类可以调用C类的方法。A类通过B类调用C类的方法。这里B是C的代理。

A通过代理B访问C ，这种模式就是代理模式。

实现代理的方式：

  • 静态代理：
    • 代理类是自己手工实现的，代理的目标是确定的
    • 实现简单、容易理解
    • 目标类增加、代理类会成倍增加；接口中的实现修改会影响很大，很多的实现类、厂家类、代理类都得修改
  • 动态代理
    • 目标类很多的时候可以使用动态代理
    • 动态指的是在程序执行的时候，调用JDK的方法才能创建代理类的对象
    • JDK动态代理：反射
    • CGLIB第三方工具库

3. Spring 循环依赖

spring 怎么处理？

  • 一级缓存：把实例化好的对象放进去
  • 二级缓存：把半实例化的对象放进去（半实例化就是刚好能注入别的对象的对象）
  • 三级缓存：把实例化对象的工厂放进去

二级缓存的重要性：

防止A->B&A->C&BC->A（A注入B，A出入C，BC注入A）这种情况下，A注入BC的时候要保证注入的A是一样的，所以要用到二级缓存，而不能用三级缓存，因为新建两个A，这两个A不一定是一样的！

为了性能

三级缓存的重要性：

三级缓存是为了当对象需要扩展或者增强的时候，可以通过工厂来生成代理对象！

为了生成代理对象

单例对象的循环依赖，spring 是可以通过缓存来解的。

但是 spring 只会提前初始化非抽象、单例 并且非懒加载的类，多例的对象是不会被加载进缓存的。

构造器注入：构造器中把对象当作参数传进去，这种被当作参数的对象是不能被添加到三级缓存中的（A想要B的实例对象的时候，B没有三级缓存所以创建不出来。同样的B想要A的时候，A也没有三级缓存，A对象也创建不出来，就会互相死锁），spring 也解决不了
