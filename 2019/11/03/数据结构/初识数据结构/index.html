初识数据结构

数据结构的预备知识，主要介绍了数据结构定义、指针、结构体、动态分配内存、跨函数使用内存原理。

先介绍几本书：

严蔚敏、吴伟民(伪算法)(清华)

高一凡(实现了上面这本书的所有伪算法)(西电)

黄国瑜(台湾人) 自己手写了一本数据结构

1. 数据结构概述

1.1. 定义

我们如何吧现实中大量而复杂的问题以特定的数据类型和特定的存储结构保存到主存储器(内存)中，以及在此基础上为实现某个功能(比如查找某个元素，删除某个元素，对所有的元素进行排序)而执行的相应操作，这个相应操作也叫算法

数据结构 = 个体的存储方式 + 个体关系的存储方式（逻辑结构+相应的操作）

算法 = 对存储数据的操作（数据结构就是想办法那现实生活中的问题储存到计算机上的过程，算法就是解决问题的步骤和方法）

广义的讲数据结构包含算法（但是有个大佬说过：程序=数据结构+算法）

1.2. 算法

算法 = 对存储数据的操作

广义算法：算法和存储关系没有关系

狭义算法：存储方式不一样，那么执行的操作就不一样

我们现在研究的是 狭义算法（我们现在研究的算法会根据存储结构的不同而不同）

算法：解题的方法和步骤

1.3. 衡量算法的标准

时间复杂度：大概程序要执行的次数，而非执行时间 (执行的机器不一样，会影响时间，执行时间依赖硬件)

空间复杂度：算法执行过程中大概所占用的最大内存(消耗内存的多少)

难易程度：尽量容易理解

健壮性： 一个问题你可能并没有考虑全面（一般指的是对于异常数据的处理能力）
这时你写了一个算法，健壮性强，就说明如果遇到你考虑之外的东西也能处理
健壮性差就说明这个算法刚刚符合你自己考虑的

1.4. 数据结构的地位

数据结构是软件中最核心的课程

程序 = 数据的存储 + 数据的操作 + 可以被计算机执行的语言

2. 预备知识

2.1. 指针

指针的重要性：指针是c语言的灵魂

2.1.1. 定义

地址：内存单元的编号。(32位)从0开始的非负整数，范围0-FFFFFFFF(0-4G-1)(这里的G和硬盘的G意义相同)

指针：指针就是地址，地址就是指针。

指针变量是存放内存单元地址的变量。

指针的本质：是一个操作受限的非负整数(不能加减乘除、只能在某些特殊情况下才能减)

指针的分类：1.基本类型的指针 2.指针和数组的关系

2.1.2. 小知识点

操作系统分配内存的过程：

​ 整个硬件是由操作系统控制。程序向操作系统申请内存，所谓分配是指系统把内存分配给程序，操作系统吧这块内存的状态修改为已占用。交还就是程序告诉操作系统不再需要这个内存空间了，操作系统吧这个内存空间的状态改为空闲。但是注意整个过程中没有对内存中的数据进行删除操作，所以所有的内存在使用之前大概率都会保存着垃圾数字。操作系统不会清空内存中遗留下来的数字，只是修改了使用权限。如果在内存状态为已占用的情况下，这块内存又被访问的话，这个访问的东西就叫病毒。

cpu访问内存：

​ cpu只能直接访问内存，不能直接访问硬盘。逻辑上来讲，cpu和内存之间进行交互有三条总线进行调节：地址线（找位置、控制数据单元）、控制线（控制读写）、数据线（数据传输）。32位的电脑，内存最大为4G，因为地址线只能控制2的32次方个数据单元，内存再大没有任何意义

​ 因为内存中的空间不会被清除，所以java中的局部变量必须初始化才能使用，Java中的属性不需要初始化就可以使用，因为他默认初始化了。不管怎么样，在使用之前必须初始化。

注意事项：

  1. 指针变量也是变量，只不过他存放的不能是内存单元的内容，只能存放内存单元的地址
  2. 普通变量前不能加*
  3. 常量和表达式前不能加&
  4. 形参也属于局部变量，指针变量统一的只占4个字节

2.2. 结构体

2.2.1. 为什么会出现结构体

为了表示一些复杂的数据，因为普通的基本类型变量无法满足要求，c语言中的结构体和java中的实体类的效果一样。

2.2.2. 什么叫结构体

结构体是用户根据自己的需要自己定义的复合数据类型。

2.2.3. 如何使用结构体

两种方式：

  1. st.sid 结构体变量名
  2. pst->sid 指针（这句话的意思是：pst所指向的结构体变量中的sid这个成员）

推荐使用第二种方式，因为第二种方式省时间，省内存

2.2.4. 小知识点

  1. 结构体变量不能加减乘除但可以相互赋值

  2. 普通结构体变量和结构体指针变量作为函数传参的问题（感受指针的方便之处，可以在传参时节省内存空间，提升程序的运行效率，减少了运行时间）

2.3. 动态内存的分配和释放

malloc函数分配内存，free函数释放内存。使用完内存之后，必须通过free释放，如果没有free的话，即使函数已经终止了，这个内存也不会释放，直到这个程序终止的时候，这个内存才有可能会被释放。但也不是绝对的，也可能会出现内存泄漏的问题。这样会导致我们的内存越来越小，这也是我们的系统越用越慢的原因，内存会变得越来越少

为什么说java很安全？

​ 在java里面不需要自己malloc和自己free，java有一个java虚拟机会自动进行回收

2.5. C++的相关概念

2.5.1. new和malloc的区别：

  1. new和delete操作配对，释放数组需要用delete[]指针。new和delete实际上调用了malloc和free，另外调用了类的构造函数和析构函数
  2. malloc和free配对，malloc返回的是void指针，需要强转，例如：（int）malloc(sizeof(int))*
  3. new申请的内存保存在堆中，malloc申请的内存保存在自由存储区

2.5.2. c++运算符

  1. 取模操作符：%
  2. 逻辑否、或、与：！&& ||
  3. 三元操作符：c = (a>b)?a:b;
  4. 按位与、或、非：& | ！
  5. 按位移：左移：<<（*左移一位相当于乘以二*）；右移：>>

2.5.3. 引用

指针,内存要为它分配一个存储空间。但是引用不分配内存空间的，引用只是一个别名。引用就好比在符号表里增加一个标志而已，对于语句int &y=x; (&x=&y)为true。

引用的一些规则：

  1. 引用被创建的同时必须被初始化(指针则可以在任何时候被初始化)，否则会报编译错误
  2. 一旦引用被初始化，就不能改变引用的关系(指针可以随时改变所指的对象)
  3. 不能有NULL引用，引用必须与合法的存储单元关联(指针可以时NULL)
  4. “sizeof引用”得到的是所指向的变量(对象)的大小，但是当引用作为成员时，其占用空间与指针相同(没找到标准的规定)
  5. 引用只能指向一个实际的变量，不能指向指针或引用
  6. 指针和引用在内部的实现其实没多大区别。但使用时有些地方是要注意的。因为引用具有对象行为，这一点很重要。引用赋值时会调用对象的复制函数，在涉及多态时，这地方很容易出错。

2.5.4. const

2.5.4.1. const作用于普通变量

注意 ！！！ const的位置不同，所起到的作用也不相同：

  1. 下面的两个声明都指向了一个const int类型的指针，指针所指向的内存不能被修改，但指针可以指向另一个内存：const int *p; int const * p;
  2. int类型的const指针应该这样声明。指针所指向的内存可以被修改，但指针不能指向另一个内存：int * const r = &n;

综上：（const int）类型的指针不能改变所指内容，而指针常量（* const）不能修改指针所指的地址

2.5.4.2. const在函数声明中的含义

第一个const：函数的返回值限定为const，即返回值不能被修改。第二个const：函数的形参为const类型，函数体内不能被修改。第三个const：表明这个函数不会对这个类对象的数据成员(非静态数据成员)作任何改变

2.5.4.3. 类的const和static成员变量的初始化

对于static成员变量，如果同时是const的，可以在类定义中初始化，否则只能在类定义外部初始化。非static的const成员变量只能在构造函数的初始化列表中初始化。

  2. long double和float变量的赋值方法：

2.5.7. 函数默认值 重载 内联 数组传参

  1. 指定函数的默认参数值： int divide(int a,int b = 2){}

  2. 什么是函数重载？(Overloaded functions)

    ​ 两个不同的函数可以用同样的名字，只要他们的参数(arguments)的原型(prototype)不同，也就是说你可以把同一个名字给多个函数，如果他们用不同数量或类型的参数

  3. 内联函数

    ​ inline指令可以被放在函数声明之前，要求该函数在被调用的地方以代码形式被编译。这相当于一个宏定义(macro)。他的好处只对短小的函数有效，这种情况下为了避免调用函数的一些常规操作时间时间(overhead)，如参数堆栈操作的时间，所以编译结果的运行代码会更快一些。调用函数的时候不需要写关键字inline，只有在函数声明前需要写

2.5.9. 类的private/protected/public属性

  1. 类成员如果没有指定访问域，默认是privatede的
  2. 标识符protected 与 private类似，他们唯一区别在继承时才表现出来。当定义一个子类的时候，基类的protected成员可以被子类的其他成员所使用，然而private成员就不可以
  3. public/protected/private继承的区别：
    • public继承：父类的public依然是public，protected依然是protected，private不可访问
    • protected继承：父类的public称为protected，protected称为private
    • private继承：父类的所有成员全部变成private
