1.3. 运算符

1.3.1. 结果是数字的运算符

一、结果是数字的运算符又分为以下两种类型：

算术运算符：+ - * / ++ –

赋值运算符：+= *= /= %= -=

二、运算结果的类型：

  1. 如果参与运算的数据类型相同，结果默认是当前数据类型：int long float double
  2. 如果参与运算的数据类型相同，如果是byte short，那么运算结果默认是 int 类型
  3. 如果参与运算的数据类型不相同，结果默认是范围比较大的数据类型：int long float double
  4. 如果参与运算的数据类型不相同，如果是byte short，那么运算结果默认是 int 类型

三、*++*

  1. 变量++：先使用变量后++
  2. ++变量：先++后使用变量

四、优先级

​ 一个表达式中出现了多个运算符，就会出现优先级的问题

​ 优先级：++ > 算数优先级 > 赋值运算符

五、字符char运算结果的类型

  1. 字符和数字直接进行计算，结果的类型可以是char可以使int，结果的类型取决于接收的类型
  2. 字符和int变量进行运算，结果默认是int类型

1.3.2. 结果是关系(boolean)的运算符

结果是关系(boolean)的运算符又分为以下两种类型：

关系运算符：> >= < <= == !=

逻辑运算符：&& || ！

注意&&和||的短路运算：前面的表达式就可以决定整个表达式的结果，后面的表达式不执行

1.3.3. 按照参与运算的个数分类的运算符

  1. 一元运算符：参与运算的数据有一个
  2. 二元运算符：参与运算的数据有一个
  3. 三元运算符：参与运算的数据有一个

结果的类型与参与运算的数据类型有关

1.5.2. 类类型

一、类类型的定义：

类类型 class 名字

类就是具有相同特征和行为的一类事物

二、类名和文件名的关系：

  1. java文件中，可以包含多个类，类名默认是文件名
  2. 一个java文件中有多个类，文件名要和public类名一致
  3. java中可以有多个类，但是只能有一个是public
  4. 文件中的类都不是public的，那么java的文件名可以任意

三、属性私有化和get、set方法：

  1. get方法 获取到属性的值，可以进行其他的操作，无参数，有返回值
  2. set方法 传入数据，将数据给属性进行赋值，有参数，无返回值，也叫访问器方法
  3. 方法的形参，当前方法生效是局部变量。属性在当前类中生效是全局变量。如果重名的话 this 代表的是当前对象即全局变量

四、构造方法：

  1. new + 构造方法 =>构造方法
  2. java中只要创建了类机会默认有一个无参数的构造方法
  3. 书写格式：public 类名(){}
  4. 只要在类中进行了构造方法的书写，那么就会覆盖默认无参数的构造方法
  5. 方法名相同，参数列表不同（个数不同、顺序不同、类型不同），这种现象叫做方法重载
  6. this() 可以进行构造方法的调用，必须放在方法体的第一句

五、类和类之间的情况

  1. 两个类的属性和方法重复，但是属性只能在类中，get与set也只能在类中。我们现在就可以把两个类相同的部分抽离到一个类中！（重复的代码放在一个类中）
  2. 之前的两个类怎么才能使用刚刚创建的类的属性和方法呢？类的继承extends（子类 extends 父类）。子类继承了父类那么子类就拥有了父类的特征（单根继承 ：一个类只能有一个父类）
  3. 创建了类就是在继承，如果没有通过extends指定的话，默认的父类就是Object（顶级父类）
  4. super可以向上调用，调用父类。只能在构造方法第一句，不能和 this 同时出现在一个构造方法中
  5. 表示数据范围大小的关系：父类的数据范围较大，子类的数据范围较大。小转大直接转换，大转小强制类型转换。变量可以调用什么方法，要看变量的类型。如果子类向上扩展（向上转型），那么会出现子类的扩展功能隐藏。动态绑定：当我们右键运行时，虚拟机会找到真正的new的对象的实际类型，执行实际类型中的方法
  6. 方法重写，子类要实现自己特殊的功能，而不是使用父类通用的功能
  7. 抽象方法：父类的方法不会执行，但是强 制子类有这个功能。抽象方法一定要出现在抽象类中

1.5.3. 接口类型

一、接口定义：

注重功能、只有方法的一个数据类型，接口中定义的是抽象方法，默认是public abstract

二、接口与实现类

  1. 定义了接口就可以用一个实现类来实现，一个实现类可以实现多个接口（多实现）
  2. 实现类和接口之间也可以进行子类和父类之间的 向上转型、动态绑定、方法重写
  3. 接口中的数据类型默认用public static final修饰，代表公共静态的常量。所以接口中定义的数据类型必须赋值才能存在

三、static

1、static：优先加载。优先加载的原理就是static修饰的数据（方法或者变量）会在类加载的时候分配内存！

JVM内存模型：

  • 方法区：存储class文件。java运行要先编译，虚拟机运行的是class文件，java文件中有什么class文件中就有什么。class文件中一般会有static main方法，所以main方法这个时候就被分配了内存，但是mian不在方法区执行，所有的方法都会在栈区中执行
  • 栈区：专门用来执行方法的区域。每个方法都会栈区中执行。每个方法都会在栈区中形成方法的栈帧。方法中的局部变量要等待，直到在堆中创建对象之后，方法中的局部变量才可以使用
  • 堆区：创建对象。new Cat()，在创建对象的时候就会分配空间，分配了空间就相当于进行了数据的存储。存储完成之后栈区的局部变量才可以调用相应的方法了

2、类方法、类变量

通过普通的对象进行方法的使用，只有创建了对象，分配了空间，才能调用了方法，执行方法。但是static修饰的数据（对象）在类加载的时候就分配了空间，类加载之后就可以调用了。static修饰的数据（对象）优先于创建对象的方式分配内存。

现在我们static修饰的数据还没有创建对象，但是有了内存空间。那我们的数据怎么调用呢？答案是通过类调用。如果调用的是方法，那么这个方法可以叫做类方法。如果调用的是变量，那么这个变量一定是全局变量，这个变量可以叫做类变量。

类方法不能被重写！

3、static不能修饰类

  • 类加载机制：惰性加载。static：优先加载。所以static不能修饰类。
  • static可以修饰方法，但是修饰的方法不能被重写。类方法不能被重写。
  • static{}静态代码块，随着了的加载自动执行，执行一次

4、static修饰 方法、全局变量、静态代码块

四、final：最终的。

  1. 必须初始化，不可以修改的数据。
  2. 可以修饰局部变量，也可以修饰全局变量。修饰全局变量的时候可以在定义变量的时候初始化，也可以在构造方法中初始化
  3. final修饰的类不能被继承。因为子类是在父类的基础上进行扩展，而final修饰的类是最终的类不能修改的类
  4. final不能修饰抽象类，不能和abstract一起使用。因为final代表的是不可变，但是abstract要子类去继承才有意义
  5. final修饰的方法不能被重写。因为子类重写父类的方法修改了父类的方法体，不能被重写
  6. final修饰 方法、局部变量、全局变量、普通类（不能修饰抽象类）

五、static+final

  1. static+final代表常量，常量名称要大写。
  2. 如果修饰基本数据类型，那么该常量的值不能更改
  3. 如果修饰的是引用数据类型，那么代表该常量的地址不能改变

1.6. 集合

一、定义

java一个变量存储多个数据

集合和数组都可以一个变量存储多个数据，但是数组一旦定义长度就不能被修改。所以出现了集合，长度可以动态的改变

二、集合的分类

  1. 单列数据存储（Collection）

    List接口：

    • 实现类ArrayList：可以存储重复的值。存储的顺序就是我们放入的顺序，是有序的集合。可以通过forEach遍历，也可以通过迭代器（Iterator）遍历
    • 实现类LinkList

    Set接口：

    • 实现类HashSet：没有下标，没有顺序（不按照书写的顺序存储）。不能存储重复值。可以通过迭代器（Iterator）进行访问

  2. 双列数据存储（Map<Key,Value>）

    • 存储的时候放入key和value，取值的时候只需要通过key来获取value
    • 如果key相同的话，后存入的值会覆盖之前存储的数据
    • 可以通过 maps.keySet() 获取 key 的值。也可以用 maps.entrySet() 进行迭代器遍历获取到的是key和value的所有值。还可以用 map.values() 进行map数组值的获取

三、泛型

什么是泛型？定义时不用指定的类型，使用时指定类型。如果没有指定的话默认就是Object，如果指定了就是指定的类型
